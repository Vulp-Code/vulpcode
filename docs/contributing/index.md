# Contributing

Vulpcode e licenciado sob MIT. Contribuicoes sao bem-vindas — bugs, novos
providers, novas tools, melhorias na documentacao.

## Roteiro

1. [Setup de desenvolvimento](dev-setup.md) — clone, venv, dependencias, rodar
   testes, lint, build do site e do pacote.
2. [Convencoes de codigo](code-conventions.md) — versao do Python, type hints,
   docstrings (Google), imports, naming, testes, commits, idioma.
3. **Adicionar provider** — passos para suportar um novo modelo. _(em
   `contributing/add-provider.md`, criado na proxima tarefa)._
4. **Adicionar tool** — passos para criar uma nova tool. _(em
   `contributing/add-tool.md`, criado na proxima tarefa)._

## Onde discutir

- **Bugs e features**: [issues do GitHub](https://github.com/vulpcode/vulpcode/issues).
- **Pull requests**: contra a branch `main`.
- **Perguntas de uso**: leia primeiro o [Guia do usuario](../user-guide/index.md)
  e a [referencia de API](../api/index.md).

## Code of conduct

O projeto adota o [Contributors Covenant](https://www.contributor-covenant.org/).
Seja respeitoso. Discussoes tecnicas devem focar no merito da ideia, nao em
quem a propos.
